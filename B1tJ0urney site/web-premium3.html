🧠 3.1 JavaScript: The Engine of Interactivity

If HTML is the skeleton and CSS is the decoration, JavaScript is the muscular and nervous system that allows the page to react to user actions.

The Pillars of Modern JS:
• ES6+ Syntax: Cleaner, more readable, and modular code.
• DOM Manipulation: The power to change HTML and CSS in real time.
• Asynchronous Programming: Load data without blocking navigation.

⚡ 3.2 Modern Syntax (ES6+)

The evolution of JavaScript brought tools that make development faster and less error-prone.

// 1. Scope Variables
const PI = 3.14; // Immutable
let counter = 0; // Can change

// 2. Arrow Functions
const greet = (name) => `Hello, ${name}!`;

// 3. Destructuring
const { title, price } = product;
  • Template Literals: Use backticks to create dynamic strings.
  • Spread Operator (...): Makes copying and merging objects or arrays easier.
  • Modules: Project organization through import and export.

🌳 3.3 DOM Manipulation

The DOM (Document Object Model) is the tree that represents your HTML. With JS, you can "bring to life" every branch of this tree.

Common Operations:
• querySelector(): Selects elements like CSS does.
• addEventListener(): Listens for clicks, scrolls, or form submissions.
• createElement(): Generates dynamic content without reloading the page.

⏳ 3.4 Asynchronous JavaScript (Fetch API)

Essential for communicating with external servers and APIs without interrupting the user experience.

async function loadData() {
  try {
    const response = await fetch('https://api.example.com');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error("Network error!");
  }
}

Using Async/Await is the most modern and readable way to handle Promises (promises of future data).

🚀 3.5 Browser APIs and Optimization

The browser offers native tools to improve performance and user experience:

  • Local Storage: Saves preferences (like Dark Theme) permanently in the browser.
  • Intersection Observer: Detects when an element appears on screen (ideal for scroll animations).
  • Debouncing: Prevents heavy functions (like searches) from running hundreds of times per second.

💡 3.6 Best Practices (Clean Code)

  • Priority: Always use const; reserve let only for counters or changing values.
  • Modularity: Create small functions that do only one thing.
  • Separation: Never use inline JS (onclick in HTML). Keep logic in the .js file.
  • Naming: Use variable names that explain what they store (e.g., isUserLoggedIn instead of x).

🔁 3.8 Arrays and Iteration

Modern JavaScript provides powerful methods to work with lists.

const prices = [10, 20, 30];

// Loop
prices.forEach(price => console.log(price));

// Transform
const discounted = prices.map(p => p * 0.9);

// Filter
const expensive = prices.filter(p => p > 15);
Methods like map, filter, and reduce are widely used in modern frameworks.

🎯 3.9 Event Handling

Events allow your application to react to user interaction.

const button = document.querySelector(".btn");

button.addEventListener("click", () => {
    alert("Button clicked!");
});
  • click
  • input
  • submit
  • scroll
  • keydown

🧩 3.10 Event Delegation

Instead of adding listeners to many elements, listen on a parent.

document.querySelector(".list").addEventListener("click", (e) => {
    if (e.target.tagName === "LI") {
        console.log("Item clicked:", e.target.textContent);
    }
});
Improves performance and simplifies dynamic content handling.

⏱ 3.11 Timers and the Event Loop

JavaScript runs on a single thread, but asynchronous tasks are handled through the event loop.

setTimeout(() => {
    console.log("Executed after 2 seconds");
}, 2000);

setInterval(() => {
    console.log("Repeating task");
}, 1000);

Store small amounts of data directly in the browser.

🔐 3.17 Scope and Closures

Scope defines where a variable can be accessed. Closures allow a function to remember variables from its outer scope.

function createCounter() {
    let count = 0;

    return function () {
        count++;
        console.log(count);
    };
}

const counter = createCounter();
counter(); // 1
counter(); // 2
Closures are widely used in modules, event handlers, and frameworks.

🧭 3.18 Understanding "this"

The value of this depends on how a function is called.

const user = {
    name: "Alice",
    greet() {
        console.log("Hello " + this.name);
    }
};

user.greet();
  • In objects: refers to the object
  • In arrow functions: inherits from the outer scope
  • In event handlers: refers to the element

📡 3.19 Working with JSON

Most APIs communicate using JSON (JavaScript Object Notation).

const jsonString = '{"name":"Alice"}';

// Convert to object
const obj = JSON.parse(jsonString);

// Convert to JSON
const json = JSON.stringify(obj);

🧪 3.20 Defensive Programming

Professional code anticipates errors and unexpected data.

  • Check if values exist before using them
  • Use optional chaining ?.
  • Use default values
const user = {};
console.log(user.profile?.name || "Guest");

🧱 3.21 Modular Code Structure

Large projects are divided into modules to improve organization and scalability.

// utils.js
export function sum(a, b) {
    return a + b;
}

// main.js
import { sum } from './utils.js';
console.log(sum(2, 3));

⚙️ 3.22 Debounce and Throttle

Used to control how often a function executes during rapid events.

  • Debounce: Runs after the user stops typing
  • Throttle: Limits execution rate (e.g., scroll events)
function debounce(fn, delay) {
    let timeout;
    return (...args) => {
        clearTimeout(timeout);
        timeout = setTimeout(() => fn(...args), delay);
    };
}

📈 3.24 Memory and Garbage Collection

  • Remove unused event listeners
  • Avoid global variables
  • Clear intervals and timeouts
  • Be careful with large objects in closures
