🌳 2.1 Git Fundamentals & Workflow

Before writing modern code, we need to know how to manage it. Git is the industry-standard tool for version control.

What you need to know:
• Git: An intelligent history log that records changes. it allows you to compare versions and recover old code if something fails.
• Workflow: A set of rules (who does what and in what order) that ensures the team doesn't overwrite each other's work.

🏗️ 2.2 HTML5: Semantic Structure

Modern HTML5 focuses on meaning (semantics) and not just appearance. This is what allows Google (SEO) to correctly index your business.

Essential Semantic Tags:
• <header>, <nav>, <main>: Base structure.
• <article>, <section>: Organization of content blocks.
• <aside>: Side content or toolbars.
• <footer>: Footer information and contact details.

📝 2.3 Forms and Multimedia

HTML5 eliminated the need for external plugins for common tasks like videos or email validation.

  • Input Types: email, date, number and range with native validation (required).
  • Multimedia: Use of <video> and <audio> with support for subtitles and Picture-in-Picture.

🎨 2.4 CSS3: The Power of Modern Design

Today, complex layouts are solved with robust alignment systems:

Layout Systems:
1. Flexbox: Ideal for aligning elements in a row or column (e.g., navigation bars).
2. CSS Grid: Perfect for two-dimensional page layouts. 
/* Example of Variable and Flexbox */
:root { --primary: #5b7cff; }
.nav { 
    display: flex; 
    background: var(--primary); 
}

📱 2.5 Responsive Design (Mobile First)

A professional website must adapt to any screen size. The focus always starts with mobile devices.

  • Media Queries: Change styles based on viewport width.
  • Relative Units: Use of rem, vw, and vh for fluid sizing.
  • Fluid Images: Ensure images never break the container layout.

✨ 2.6 Performance and BEM Architecture

Organizing CSS is as important as design. The BEM pattern helps keep the code clean:

BEM Example:
• .card (Block)
• .card__title (Internal Element)
• .card--highlighted (Modificador/Variante)

For performance: Minimize CSS, optimize images for WebP and use Lazy Loading.

✅ 2.7 Conclusion: Are you ready for Frameworks?

Mastering semantic HTML, modern layouts (Flex/Grid), and responsive design is a mandatory requirement before moving on to React, Vue, or Angular.

Next step: Go to the Lab and try creating a card using the new semantic tags and Flexbox!

🔀 2.8 Branching and Collaboration with Git

In professional environments, developers do not work directly on the main branch.

  • main / master: Stable production code
  • feature branches: New features or fixes
  • merge: Combine changes into main
  • pull request: Code review before integration
git checkout -b feature/login
git add .
git commit -m "Add login feature"
git push origin feature/login

☁️ 2.9 GitHub and Remote Repositories

GitHub (or GitLab/Bitbucket) allows teams to collaborate and store code remotely.

  • Version backup in the cloud
  • Issue tracking and project management
  • Code reviews with Pull Requests
  • Portfolio for developers
git remote add origin https://github.com/user/project.git
git push -u origin main

🌐 2.10 HTML5 APIs (Modern Capabilities)

HTML5 introduced powerful browser features without external plugins.

  • Local Storage: Store data in the browser
  • Geolocation: Access user location (with permission)
  • Canvas: Draw graphics and animations
  • Drag and Drop: Interactive UI behavior

🎥 2.11 Advanced Multimedia Optimization

  • Provide multiple video formats (mp4, webm)
  • Use subtitles with <track>
  • Compress media to reduce load time
  • Lazy load heavy content when possible

🎨 2.12 CSS3 Advanced Positioning

  • relative: Position relative to itself
  • absolute: Position inside a parent container
  • fixed: Stays fixed on screen
  • sticky: Switches between relative and fixed on scroll

✨ 2.13 Transitions and Animations

Modern interfaces use motion to improve user experience.

.button {
    transition: transform 0.3s ease, box-shadow 0.3s ease;
}

.button:hover {
    transform: translateY(-3px);
    box-shadow: 0 5px 15px rgba(0,0,0,0.2);
}

🎛️ 2.14 CSS Variables and Theming

Custom properties allow dynamic and maintainable design systems.

:root {
    --primary: #5b7cff;
    --spacing: 16px;
}

.card {
    padding: var(--spacing);
    border-color: var(--primary);
}

📐 2.15 Modern CSS Units

  • rem – Scalable typography
  • vw / vh – Viewport-based layouts
  • clamp() – Responsive values without media queries
h1 {
    font-size: clamp(1.5rem, 4vw, 3rem);
}

⚡ 2.16 Modern Performance Practices

  • Use WebP or AVIF images
  • Load CSS efficiently
  • Avoid large layout shifts
  • Reduce unused CSS
  • Optimize fonts and use font-display

🧠 2.17 Professional Workflow Mindset

  • Work with Git branches for every feature
  • Write meaningful commit messages
  • Test responsiveness continuously
  • Keep CSS modular and reusable
  • Think in components, not pages

🔎 2.19 Git Internals: How Git Actually Works

Git is not just a file tracker — it is a snapshot-based version system.

  • Each commit is a snapshot of the project
  • Commits are linked using hashes (SHA-1)
  • Branches are lightweight pointers
  • HEAD represents your current working state
git log --oneline --graph --all

🧨 2.20 Merge vs Rebase

Understanding this difference separates junior from professional developers.

  • Merge: Combines histories, keeps branch structure
  • Rebase: Rewrites commit history for a cleaner timeline
  • Rebase is powerful but must be used carefully
git rebase main
git merge feature/login

📦 2.21 Git Best Practices

  • Write clear commit messages (imperative tone)
  • Commit small logical changes
  • Never commit sensitive data
  • Use .gitignore properly
  • Pull before pushing

🌍 2.22 Progressive Enhancement

Build the basic experience first, then enhance with modern features.

  • Core content must work without JavaScript
  • Enhance interaction gradually
  • Ensure accessibility first

🎨 2.23 CSS Grid Advanced Layout

.layout {
    display: grid;
    grid-template-areas:
        "header header"
        "sidebar main"
        "footer footer";
    grid-template-columns: 250px 1fr;
}

Grid allows magazine-level layout control with minimal code.

🧩 2.24 CSS Specificity & Cascade

Understanding the cascade prevents styling conflicts.

  • Inline styles > IDs > Classes > Elements
  • Avoid !important in professional code
  • Keep selectors predictable and shallow

🌈 2.25 Modern Visual Effects

  • Gradients
  • Backdrop filters
  • Glassmorphism
  • Neumorphism (used carefully)
  • Advanced shadows
.glass {
    backdrop-filter: blur(10px);
    background: rgba(255,255,255,0.1);
}

⚡ 2.26 Critical Rendering Path

Understanding how the browser renders pages is key for performance.

  • HTML parsing
  • CSSOM construction
  • Render tree creation
  • Layout calculation
  • Painting & compositing

📱 2.27 Container Queries (Modern CSS)

Unlike media queries, container queries respond to parent size.

@container (min-width: 400px) {
    .card {
        flex-direction: row;
    }
}

🎯 2.28 Design Systems Thinking

Professional projects use reusable components.

  • Define color tokens
  • Define spacing scale
  • Reusable buttons, cards, inputs
  • Consistent typography scale

🔐 2.29 Security Basics in Front-End

  • Never trust user input
  • Prevent XSS vulnerabilities
  • Use HTTPS always
  • Sanitize dynamic content

📊 2.30 Lighthouse & Performance Audits

Use browser tools to measure:

  • Performance score
  • Accessibility score
  • SEO score
  • Best Practices
