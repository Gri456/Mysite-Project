🎨 2.1 CSS Custom Properties (Variables) — In Depth

CSS Custom Properties (often called CSS Variables) allow you to store values in a reusable, dynamic way. Unlike Sass variables (which are resolved at compile time), CSS variables are live in the browser — they can be changed at runtime via JavaScript and cascade through the DOM.

/* Define on :root to be available everywhere */
:root {
  --color-primary:    #5b7cff;
  --color-secondary:  #28e0ff;
  --spacing-sm:  0.5rem;
  --spacing-md:  1rem;
  --spacing-lg:  2rem;
  --radius:      0.75rem;
  --shadow:      0 4px 20px rgba(0, 0, 0, 0.2);
}

/* Use with var() */
.card {
  padding:       var(--spacing-md);
  border-radius: var(--radius);
  box-shadow:    var(--shadow);
  color:         var(--color-primary);
}

/* Override for a specific scope */
.dark-section {
  --color-primary: #fff;  /* Only affects children of this element */
}

/* Change via JavaScript for theme switching */
document.documentElement.style.setProperty('--color-primary', '#ff6b6b');
Why this matters for professional projects: CSS variables are the foundation of every design system and theming architecture. They let you build light/dark mode in seconds, maintain brand consistency, and change an entire UI by editing a handful of values.

🔍 2.2 Advanced CSS Selectors

Beyond basic tag, class and ID selectors, CSS has a rich set of selectors that let you target elements with surgical precision — without extra classes or JavaScript.

/* Attribute selectors */
input[type="email"]      { border-color: var(--cyan); }
a[href^="https"]         { color: green; }  /* starts with */
a[href$=".pdf"]          { color: red; }    /* ends with */
a[href*="docs"]          { font-weight: bold; } /* contains */

/* Structural pseudo-classes */
li:first-child            { font-weight: bold; }
li:last-child             { border-bottom: none; }
li:nth-child(2n)          { background: rgba(255,255,255,0.04); } /* even rows */
li:nth-child(3n+1)        { color: var(--cyan); }
p:not(.intro)             { margin-top: 1rem; }

/* Relationship selectors */
.parent > .child          { } /* direct child only */
h2 + p                    { } /* immediately after sibling */
h2 ~ p                    { } /* all following siblings */

/* State pseudo-classes */
button:hover              { transform: translateY(-2px); }
input:focus               { outline: 2px solid var(--blue); }
input:invalid             { border-color: red; }
input:valid               { border-color: green; }
:is(h1, h2, h3)           { line-height: 1.2; } /* matches any */
:where(section, article)  { padding: 2rem; }    /* zero specificity */
a:has(img)                { display: block; }   /* parent selector! */
:has() is the most powerful new selector — it lets you style a parent based on its children. Finally supported in all modern browsers (2023+).

✨ 2.3 CSS Transitions vs Animations

CSS has two mechanisms for motion: transitions (from A to B, triggered by state change) and animations (full keyframe control, runs automatically).

/* ── TRANSITIONS ─────────────────────────────────── */
/* Format: property duration timing-function delay */
.button {
  background: #5b7cff;
  transform: scale(1);
  transition: background 0.3s ease,
              transform 0.2s cubic-bezier(0.34, 1.56, 0.64, 1); /* spring! */
}
.button:hover {
  background: #28e0ff;
  transform: scale(1.06);
}

/* ── KEYFRAME ANIMATIONS ──────────────────────────── */
@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

@keyframes pulse {
  0%, 100% { transform: scale(1); }
  50%       { transform: scale(1.05); }
}

.hero-title {
  animation: fadeInUp 0.6s ease forwards;
}
.notification-dot {
  animation: pulse 2s ease-in-out infinite;
}

/* Staggered entrance — children appear one by one */
.card:nth-child(1) { animation-delay: 0.0s; }
.card:nth-child(2) { animation-delay: 0.1s; }
.card:nth-child(3) { animation-delay: 0.2s; }

/* Respect user preferences (accessibility!) */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

📐 2.5 Advanced CSS Grid

CSS Grid is the most powerful layout tool on the web. These are the patterns that separate beginners from professionals.

/* Named template areas — self-documenting layouts */
.layout {
  display: grid;
  grid-template-areas:
    "header  header  header"
    "sidebar content aside"
    "footer  footer  footer";
  grid-template-columns: 200px 1fr 160px;
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
}
header  { grid-area: header;  }
.sidebar{ grid-area: sidebar; }
main    { grid-area: content; }
aside   { grid-area: aside;   }
footer  { grid-area: footer;  }

/* Responsive without media queries! */
.auto-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 1.5rem;
}

/* Spanning cells */
.featured { grid-column: span 2; grid-row: span 2; }

/* Dense packing (fill holes automatically) */
.masonry { grid-auto-flow: dense; }

/* Subgrid — children align to parent grid */
.card {
  display: grid;
  grid-row: span 3;
  grid-template-rows: subgrid; /* Chrome 117+, Firefox 71+ */
}

/* Placing items explicitly */
.item { grid-column: 2 / 4; grid-row: 1 / 3; }

🌊 2.6 CSS Clamp, Min & Max — Fluid Typography

Modern CSS lets you create layouts that scale smoothly between viewport sizes without any media queries, using mathematical functions.

/* clamp(MIN, IDEAL, MAX) — scales fluidly */
h1 {
  font-size: clamp(1.5rem, 4vw, 3rem);
  /* Min: 1.5rem on tiny screens
     Max: 3rem on wide screens
     Scales proportionally between */
}

.container {
  width: clamp(320px, 90%, 1200px);
  padding: clamp(1rem, 3vw, 3rem);
}

/* min() / max() — always pick the smaller/larger value */
.sidebar {
  width: min(300px, 30%);   /* won't exceed 300px */
}
.hero {
  height: max(400px, 50vh); /* at least 400px */
}

/* Fluid spacing with calc() */
.section {
  margin-top: calc(2rem + 1vw);  /* grows with viewport */
  padding: calc(var(--spacing-md) * 2);
}

/* Responsive type scale — no breakpoints needed */
:root {
  --step-0: clamp(1rem, 0.95rem + 0.24vw, 1.125rem);
  --step-1: clamp(1.25rem, 1.14rem + 0.54vw, 1.5rem);
  --step-2: clamp(1.56rem, 1.35rem + 1.05vw, 2rem);
  --step-3: clamp(1.95rem, 1.59rem + 1.82vw, 2.66rem);
}

🎭 2.7 Pseudo-Elements ::before & ::after

Pseudo-elements inject virtual elements into the DOM through CSS alone. They are essential for decorative effects, tooltips, custom checkboxes, and complex styling without extra HTML.

/* Always need content (even empty string) */
.section-title::before {
  content: '';
  display: block;
  width: 40px;
  height: 4px;
  background: var(--cyan);
  margin-bottom: 0.5rem;
  border-radius: 2px;
}

/* Add icons before links */
.external-link::after {
  content: ' ↗';
  font-size: 0.8em;
  color: var(--blue);
}

/* Quote marks */
blockquote::before { content: '\201C'; font-size: 4rem; color: var(--blue); }
blockquote::after  { content: '\201D'; font-size: 4rem; color: var(--blue); }

/* Gradient overlay on images without extra HTML */
.hero {
  position: relative;
}
.hero::after {
  content: '';
  position: absolute;
  inset: 0;
  background: linear-gradient(to bottom, transparent 40%, rgba(0,0,0,0.8));
}

/* Custom checkbox with ::before */
.checkbox-custom::before {
  content: '';
  display: inline-block;
  width: 18px; height: 18px;
  border: 2px solid var(--blue);
  border-radius: 4px;
  margin-right: 0.5rem;
}

🔧 2.8 CSS Specificity & the Cascade — Mastered

Specificity is the algorithm browsers use to decide which CSS rule wins when multiple rules target the same element. Misunderstanding it causes debugging nightmares.

/* Specificity score: (inline, ID, class/attr/pseudo, element) */

p                { color: black; }    /* 0,0,0,1 */
.text            { color: blue;  }    /* 0,0,1,0 — wins over p */
#main            { color: green; }    /* 0,1,0,0 — wins over class */
style="color:red"                     /* 1,0,0,0 — wins over all */
!important                            /* Overrides everything — AVOID */

/* :is() and :where() for controlled specificity */
:is(h1, h2, h3) { line-height: 1.2; } /* takes highest specificity of its args */
:where(h1, h2, h3) { line-height: 1.2; } /* always 0 specificity — easy to override */

/* Cascade layers — explicit ordering (CSS 2022) */
@layer reset, base, theme, components, utilities;

@layer reset {
  *, *::before, *::after { box-sizing: border-box; }
  body { margin: 0; }
}
@layer theme {
  :root { --primary: #5b7cff; }
}
@layer utilities {
  .mt-auto { margin-top: auto; }
}
/* Layers declared later beat layers declared earlier
   Styles outside layers beat all layers */

🌈 2.9 Modern CSS: Container Queries, Scroll Animations & More

CSS is evolving fast. These are the 2023–2024 features that are now production-ready and changing how we write CSS.

/* ── CONTAINER QUERIES ── */
/* A component responds to its CONTAINER size, not the viewport */
.card-wrapper {
  container-type: inline-size;
  container-name: card;
}
@container card (min-width: 400px) {
  .card { flex-direction: row; }  /* horizontal when container is wide */
}

/* ── SCROLL-DRIVEN ANIMATIONS ── */
@keyframes reveal {
  from { opacity: 0; transform: translateY(30px); }
  to   { opacity: 1; transform: translateY(0);     }
}
.section {
  animation: reveal linear both;
  animation-timeline: view();        /* plays as element enters viewport */
  animation-range: entry 0% entry 30%;
}

/* ── CSS NESTING (now native, no Sass needed) ── */
.card {
  padding: 1rem;
  border-radius: 8px;

  & .title {          /* & = .card .title */
    font-size: 1.2rem;
  }
  &:hover {           /* & = .card:hover */
    background: rgba(255,255,255,0.1);
  }
  @media (max-width: 768px) {
    padding: 0.75rem; /* Nested media query! */
  }
}

/* ── COLOR-MIX ── */
.muted {
  color: color-mix(in srgb, var(--primary) 60%, transparent);
}

🎨 2.14 Customising Bootstrap with SCSS BS5

Bootstrap is built with Sass. The professional way to use Bootstrap is to override its variables before it compiles — giving you a fully branded design system that still feels like Bootstrap under the hood.

/* custom.scss — ALWAYS import your overrides BEFORE Bootstrap */

// 1. Override Bootstrap Sass variables
$primary:    #5b7cff;
$secondary:  #28e0ff;
$success:    #22c55e;
$font-family-base: 'Inter', sans-serif;
$border-radius:    0.75rem;
$box-shadow:       0 4px 20px rgba(0,0,0,.15);

// 2. Optional: Disable components you don't need (reduces bundle size)
$enable-gradients:  true;
$enable-shadows:    true;
$enable-cssgrid:    true;  // Bootstrap's experimental CSS Grid mode

// 3. Import Bootstrap
@import "bootstrap/scss/bootstrap";

// 4. Add your custom styles AFTER
.navbar {
  border-bottom: 1px solid rgba(255,255,255,.1);
}

/* ── Using Bootstrap's own mixins ──────────────── */
.custom-breakpoint {
  @include media-breakpoint-up(lg) {
    display: flex;
  }
  @include media-breakpoint-down(md) {
    padding: 1rem;
  }
}

.my-button {
  @include button-variant($primary, $primary); /* Generate full button states */
}

⚡ 2.16 Bootstrap JavaScript API BS5

Every Bootstrap component can be controlled programmatically — no jQuery needed.

// Programmatic control with vanilla JS

// Modal
const modal = new bootstrap.Modal('#myModal', {
  backdrop: 'static', // prevents closing on outside click
  keyboard: false
});
modal.show();
modal.hide();
modal.toggle();

// Toast notifications
const toastEl = document.getElementById('myToast');
const toast = new bootstrap.Toast(toastEl, { delay: 3000 });
toast.show();

// Tooltip (must be initialised — they're opt-in)
const tooltips = document.querySelectorAll('[data-bs-toggle="tooltip"]');
tooltips.forEach(el => new bootstrap.Tooltip(el));

// Popover
new bootstrap.Popover('#btn', {
  title: 'Info',
  content: 'This is a popover.',
  placement: 'top'
});

// Offcanvas sidebar
const offcanvas = bootstrap.Offcanvas.getOrCreateInstance('#sidebar');
offcanvas.show();

// Listen for Bootstrap events
document.getElementById('myModal').addEventListener('shown.bs.modal', () => {
  console.log('Modal is now fully visible');
});
document.getElementById('myModal').addEventListener('hidden.bs.modal', () => {
  console.log('Modal has been closed');
});
