Why Logic & Reasoning Matter in AI

Before Machine Learning, Neural Networks, and LLMs existed, AI relied on logic, symbolic reasoning, and search algorithms. These foundations are still essential today because they explain:

  • How machines make decisions step by step
  • How intelligent agents perceive and act in environments
  • How to solve problems algorithmically without training data
  • How pathfinding and planning work (GPS, games, robotics)
  • How rules, constraints, and knowledge are represented formally

Even modern AI systems use these concepts — especially in planning, robotics, navigation, and game AI. A* pathfinding still powers GPS apps today. FOL underpins knowledge graphs used by Google.

There are two major paradigms in AI that have competed and coexisted throughout history:

Paradigm 1
Symbolic AI (GOFAI)
"Good Old-Fashioned AI" — knowledge represented as symbols, rules, and logic. Explainable and auditable, but brittle when facing noisy real-world data. This module covers this paradigm.
Paradigm 2
Connectionist AI (ML/DL)
Knowledge emerges from statistical patterns learned from data. Powerful and flexible, but often opaque ("black box"). This is the basis of modern neural networks and LLMs.
Paradigm 3
Neuro-Symbolic AI
A growing hybrid approach combining deep learning with symbolic reasoning — combining the strengths of both paradigms. Active research area at DeepMind, IBM, and MIT.
📌 Why study classical AI in the age of LLMs?

Modern LLMs cannot guarantee correctness, plan reliably, or reason about formal constraints. Symbolic methods are used alongside LLMs in robotics (motion planning), compilers (constraint solving), logistics (scheduling), and safety-critical systems where correctness must be verified.

Intelligent Agents – The Core of Classical AI

An intelligent agent is anything that perceives its environment, makes decisions, takes actions, and tries to maximize performance.

Examples: Self-driving car · Robot vacuum · Chess engine · NPC in a game · Search engine

Sensors — what the agent perceives (camera, microphone, game state)
Actuators — what the agent does (move, speak, click, API call)
Agent Function / Policy — the logic mapping states → actions

Five types of agents:

Simple Reflex
Reacts to conditions using rules.
IF obstacle THEN turn-left
Model-Based
Maintains internal state about the environment over time.
Goal-Based
Selects actions to reach a specific goal state.
Utility-Based
Chooses the "best" action by scoring outcomes with a utility function.
Learning Agent
Improves behavior over time through experience and feedback.

Environments also vary in complexity — and the type of environment determines which agent design is appropriate:

Problem Solving & Search – How AI Finds Solutions

Early AI focused on solving problems by exploring a state space — all possible configurations of a problem.

State Space Representation:
States → all possible configurations
Initial state → where we start
Goal state → what we want to reach
Actions → what moves us from one state to another

Example: A maze — each cell is a state, moving up/down/left/right are actions. A search algorithm explores the maze to find a path from entrance to exit.

A well-defined search problem has five formal components:

Component 1
Initial State
The starting configuration. In a 8-puzzle: the scrambled tile arrangement. In navigation: your current GPS position.
Component 2
Actions
The set of possible moves from any given state. Must be defined precisely — e.g., "slide tile left" only if there's an empty space.
Component 3
Transition Model
Defines the result of each action: RESULT(state, action) → new_state. This is what builds the state space graph.
Component 4
Goal Test
A function that checks if the current state is the goal. Can be an exact match or a condition (e.g., "all tiles in order").
Component 5
Path Cost
A numeric value for each path. Used to find the cheapest solution — e.g., distance in km, time in minutes, or uniform cost 1 per step.
🔁 How search algorithms work in practice

All search algorithms maintain a frontier (nodes to explore next) and an explored set (nodes already visited). The key difference between algorithms is how they order the frontier: BFS uses a queue (FIFO), DFS uses a stack (LIFO), UCS uses a priority queue by cost, A* uses a priority queue by f(n) = g(n) + h(n).

Uninformed Search Algorithms (Blind Search)

These algorithms explore without any extra knowledge about the problem — they only know the structure of the state space.

Breadth-First Search (BFS) — explores all nodes level by level.
Guarantees the shortest path in unweighted graphs.
Uses: social networks, shortest path, peer-to-peer networks.

Depth-First Search (DFS) — explores deep before exploring wide.
Fast and memory-efficient, but can get stuck in infinite branches.
Uses: solving mazes, tree traversal, checking graph connectivity.

Uniform Cost Search (UCS) — finds the cheapest path when each step has a different cost.
Expands the lowest-cost node first. Optimal when all step costs are positive.

AlgorithmComplete?Optimal?MemoryTime
BFSYesYes (unweighted)High (all nodes)O(bd)
DFSMaybeNoLow (one path)O(bm)
UCSYesYesHighO(bd)

b = branching factor, d = depth of solution, m = maximum depth

Two important variants that address specific limitations:

Iterative Deepening DFS (IDDFS) — combines BFS's completeness and optimality with DFS's low memory footprint. It runs DFS repeatedly with increasing depth limits (1, 2, 3…) until it finds the goal. Counter-intuitively, the repeated work is negligible because tree size grows exponentially — the last iteration dominates.
Memory: O(bd) — same as DFS. Optimal: Yes (like BFS). This is often the preferred uninformed algorithm in practice.

Bidirectional Search — runs two simultaneous searches: one from the start forward, one from the goal backward, until they meet in the middle. Complexity drops from O(bd) to O(bd/2), which is a massive improvement.
Used in: advanced GPS routing, social network connection search (e.g., LinkedIn "degrees of connection").

Informed Search — Heuristics & A*

Informed search uses heuristics — estimated guesses about how close we are to the goal — to search more efficiently.

Greedy Best-First Search — uses only the heuristic h(n). Very fast, but not guaranteed to find the optimal path. Greedy means it always goes toward what looks closest to the goal.

A* Search — The King of Pathfinding
The most widely used search algorithm. Combines actual path cost with heuristic estimate:

f(n) = g(n) + h(n)

g(n) = cost from start to current node (actual path cost so far)
h(n) = heuristic estimate from current node to goal
f(n) = total estimated cost through this node

A* is optimal when the heuristic is admissible — it never overestimates the true cost to the goal.

Real-world uses of A*: Video game AI pathfinding · GPS route calculation · Robot navigation · Route optimization logistics

Choosing the right heuristic is critical — it directly determines A*'s speed and accuracy:

Heuristic Type
Manhattan Distance
|x₁-x₂| + |y₁-y₂| — counts horizontal + vertical steps. Perfect for grid-based movement where diagonal is not allowed (tile puzzles, city blocks).
Heuristic Type
Euclidean Distance
√((x₁-x₂)² + (y₁-y₂)²) — straight-line distance. Better when movement is unrestricted (open terrain, flying drones, Euclidean space).
Heuristic Type
Chebyshev Distance
max(|x₁-x₂|, |y₁-y₂|) — allows 8-directional movement. Used in chess (king moves) and grid games where diagonal steps are allowed.
Key Property
Admissibility
A heuristic is admissible if it never overestimates the true cost. All three distance metrics above are admissible in their respective contexts.
⚡ A* Variants for Large-Scale Problems
  • IDA* (Iterative Deepening A*) — A* with depth-limited search to reduce memory. Used in chess engines and robotics.
  • Weighted A* — uses f(n) = g(n) + w·h(n) where w > 1, trading optimality for speed. Used in real-time games.
  • D* (Dynamic A*) — recalculates paths as the environment changes in real time. Used in self-driving cars and Mars rovers.

Local Search & Optimization

Not all problems require finding a path — sometimes we just need to find the best configuration from a vast space of possibilities. Local search algorithms work by starting with a solution and iteratively improving it.

Hill Climbing — start with a random solution and move to the best neighboring state. Simple and fast, but vulnerable to getting stuck in local maxima (good but not globally best solutions) and plateaus (flat regions with no improvement).

Simulated Annealing — like hill climbing, but occasionally accepts worse solutions with a probability that decreases over time (the "temperature"). This allows escaping local maxima. Inspired by the annealing process in metallurgy.
Used in: chip design, protein folding, traveling salesman problem, logistics optimization.

Genetic Algorithms — inspired by evolution. A population of candidate solutions is evolved through selection, crossover (combining two solutions), and mutation over many generations. Excellent for large, complex search spaces.
Used in: engineering design, hyperparameter optimization, scheduling, game strategy evolution.

🌐 Local vs Global Search

BFS/DFS/A* are systematic — they explore the full state space and guarantee finding a solution. Local search is incomplete — it may get stuck — but it can handle huge spaces (like optimizing millions of parameters) where systematic search is computationally impossible. Modern ML training (gradient descent) is a form of local search.

Adversarial Search — Game-Playing AI

In adversarial settings (games with opponents), the agent must plan not just for its own moves but also account for an opponent trying to minimize its score.

Minimax Algorithm — models two-player zero-sum games. The AI (MAX player) tries to maximize its score; the opponent (MIN player) tries to minimize it. The algorithm builds a complete game tree and selects the move that leads to the best worst-case outcome.
Used in: chess, checkers, tic-tac-toe, Go.

minimax(state) = max over actions of min(minimax(result(state, action)))

Alpha-Beta Pruning — a major optimization of minimax. It prunes branches that cannot possibly influence the final decision, reducing the number of nodes evaluated from O(bd) to O(bd/2) in the best case — effectively doubling the searchable depth for the same computation.
Used in: all serious chess engines. Deep Blue used alpha-beta with custom evaluation functions.

Monte Carlo Tree Search (MCTS) — instead of exhaustive search, MCTS uses random simulations to estimate the value of moves. It has four phases: Selection → Expansion → Simulation → Backpropagation.
Used in: AlphaGo, AlphaZero, and games where the state space is too large for minimax.

♟️ Evolution of Game AI
  • Deep Blue (1997) — minimax + alpha-beta pruning + hand-crafted evaluation. Defeated Kasparov in chess.
  • AlphaGo (2016) — MCTS + deep neural networks for position evaluation. First AI to defeat a world Go champion.
  • AlphaZero (2017) — purely self-play reinforcement learning. Mastered chess, Go, and shogi in hours with zero human knowledge.

Logic in AI – The Origin of Machine Reasoning

Before ML, AI used symbolic logic to represent and reason about knowledge. Logic is still used in knowledge graphs, expert systems, and robotics planning.

Propositional Logic — statements are either TRUE or FALSE. Rules take the form:
IF A AND B THEN C
IF rain THEN take_umbrella
Used in: expert systems, automated testing, game logic, compliance rules.

First-Order Logic (FOL) — extends propositional logic with:
Variables (objects) · Predicates (properties/relations) · Quantifiers (∀ = "for all", ∃ = "there exists")

∀x Human(x) → Mortal(x)
Human(Socrates)
∴ Mortal(Socrates)

AI uses FOL for: Knowledge graphs · Semantic search · Robotics planning · Automated reasoning

Key logical connectives and their meanings in AI rule systems:

SymbolNameMeaningExample
∧AND (Conjunction)Both must be trueRain ∧ Cold → WearCoat
∨OR (Disjunction)At least one must be trueTired ∨ Sick → StayHome
¬NOT (Negation)Inverts the truth value¬Rain → Sunny
→ImpliesIf left is true, right followsHuman(x) → Mortal(x)
↔BiconditionalTrue when both sides matchDay ↔ ¬Night
∀For AllStatement applies to every object∀x Dog(x) → Animal(x)

🔍 Inference Methods in Logic
  • Modus Ponens — the most fundamental rule: if P → Q and P is true, then Q is true. Used in every rule-based system.
  • Resolution — a complete inference algorithm for FOL. Converts statements to clausal form and derives new facts. Basis of Prolog and automated theorem provers.
  • Unification — the process of finding variable substitutions that make two FOL expressions identical. Central to logic programming and Prolog.

Constraint Satisfaction Problems (CSPs)

A CSP is a problem where you must assign values to variables such that all constraints are satisfied simultaneously.

Classic examples:

  • Sudoku — each row, column, and box must contain digits 1–9 exactly once
  • Map coloring — adjacent regions must have different colors
  • University scheduling — no professor in two classes at once
  • Resource allocation — tasks assigned without conflicts

Solving techniques:
Backtracking — try assignments, undo if they fail, try another
Forward Checking — remove impossible values early as assignments are made
AC-3 Arc Consistency — propagate constraints to eliminate values before search begins

Two key strategies that dramatically improve CSP performance:

Minimum Remaining Values (MRV) Heuristic — when choosing which variable to assign next, always choose the one with the fewest remaining legal values in its domain. This "fail-first" strategy detects dead ends early, reducing wasted computation.
Example: In Sudoku, fill cells with only one possible value first before tackling cells with many options.

Least Constraining Value (LCV) Heuristic — when choosing which value to assign, pick the one that rules out the fewest choices for neighboring variables. This keeps options open for future assignments.
MRV chooses which variable — LCV chooses which value. Together they can reduce solve time by orders of magnitude.

🏭 CSPs in Industry
  • Operations Research — airline scheduling, crew assignment, supply chain logistics — all formulated as CSPs with millions of constraints.
  • Chip Design (EDA) — placing billions of transistors on a chip while satisfying electrical and thermal constraints.
  • Configuration Systems — SAP and Oracle use constraint engines to generate valid product configurations from customer requirements.
  • Automated Planning (STRIPS) — robotics and game AI use CSP-like planners to sequence actions toward goals.
