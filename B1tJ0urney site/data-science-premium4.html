🔍 4.1 Data Preparation & EDA — The Heart of Every Project

After collecting and loading your data, the instinct is to jump straight into building a model. Resist that instinct. The most common cause of failed Data Science projects is not a bad algorithm — it is poorly understood data fed into a good algorithm.


This phase — Data Preparation and Exploratory Data Analysis (EDA) — is where you become a detective. You interrogate your data, question its assumptions, expose its flaws, and ultimately understand what story it tells. Only then are you ready to model.

🧹
Data Cleaning

Fix errors, nulls, types, duplicates, and inconsistencies

📊
Descriptive Stats

Summarize distributions, central tendency, spread, and shape

📈
Visualization

Charts that reveal what numbers alone cannot communicate

🔗
Relationships

Discover correlations and patterns that guide feature engineering

🕵️ EDA as Investigation:
1. Understand
What does each column mean? What is the unit? What is the expected range?
2. Question
Why does this column have 30% nulls? Are these customers inactive or just unrecorded?
3. Discover
Is there a pattern? A trend? A suspicious cluster? An unexpected correlation?
4. Decide
Based on evidence, make and document cleaning and engineering decisions.

🗺️ 4.2 The EDA Workflow — A Structured Approach

EDA is not random exploration. The best Data Scientists follow a systematic structure that ensures nothing is missed before modeling. Here is the professional workflow:

Step 1 — Load & Audit

Check shape, dtypes, memory usage, head/tail. Get a feel for what you have before touching anything.

df.info() · df.describe() · df.head()
Step 2 — Univariate Analysis

Examine each column in isolation. Distributions, value counts, ranges. Identify anomalies early.

histograms · value_counts() · boxplots
Step 3 — Bivariate Analysis

Explore relationships between pairs of variables. Correlations, group comparisons, scatter plots.

scatter · corr() · groupby + boxplot
Step 4 — Multivariate Analysis

Look at multiple variables together. Heatmaps, pair plots, dimensionality reduction (PCA).

heatmap · pairplot · PCA
Step 5 — Clean & Document

Apply cleaning decisions with justification. Every change must be documented — not just what you did, but why.

fillna · drop · astype · .copy()
Step 6 — Insights & Hypotheses

Summarize discoveries. Formulate hypotheses that will guide feature engineering and model selection.

markdown cells · visualization summaries

📐 4.3 Descriptive Statistics — Summarizing Data Mathematically

Before drawing any chart, descriptive statistics give you a quantitative summary of your data. They describe where data is centred, how spread out it is, and what shape the distribution takes.

📍 Central Tendency

Where is the "middle" of the data?

  • Mean: Sum ÷ count. Sensitive to outliers.
  • Median: Middle value. Robust to outliers.
  • Mode: Most frequent value. Useful for categories.
📏 Dispersion / Spread

How spread out is the data?

  • Variance: Average squared deviation from mean.
  • Std Dev (σ): Square root of variance. Same unit as data.
  • IQR: Q3 − Q1. Range of the middle 50%.
  • Range: Max − Min. Sensitive to outliers.
📊 Distribution Shape

What shape does the data take?

  • Skewness: Asymmetry. Positive = right tail.
  • Kurtosis: Tail heaviness vs normal distribution.
  • Percentiles: Q1=25%, Q2=50%, Q3=75%.

DESCRIPTIVE STATS IN PYTHON

import pandas as pd
import numpy as np

# One-line full summary
df.describe()
# Output: count, mean, std, min, 25%, 50%, 75%, max for each numeric column

# Individual statistics
df["salary"].mean()        # Arithmetic mean
df["salary"].median()      # Median (50th percentile)
df["salary"].mode()[0]     # Most frequent value
df["salary"].std()         # Standard deviation
df["salary"].var()         # Variance
df["salary"].quantile(0.25) # Q1 — 25th percentile
df["salary"].quantile(0.75) # Q3 — 75th percentile

# IQR (Inter-Quartile Range)
Q1 = df["salary"].quantile(0.25)
Q3 = df["salary"].quantile(0.75)
IQR = Q3 - Q1
print(f"IQR: {IQR:.2f}")

# Distribution shape
df["salary"].skew()        # Skewness (0 = symmetric, +/- = skewed)
df["salary"].kurt()        # Kurtosis (3 = normal, >3 = heavy tails)

# Extend describe to all columns including categoricals
df.describe(include="all")

# For a categorical column
df["category"].value_counts()            # Count per category
df["category"].value_counts(normalize=True)  # Proportions (%)
df["category"].nunique()                 # Number of unique values
⚠️ Mean vs Median — Why it matters:

Imagine 10 employees earning €2,000/month and one CEO earning €500,000. The mean salary would be ~€47,000 — completely misleading. The median would be €2,000 — the true picture. Always compare mean and median. If they differ significantly, the distribution is skewed and the mean is not representative.

🎯 4.5 Outlier Detection and Treatment

An outlier is a data point that differs significantly from the rest of the data. But not all outliers are equal — some are errors to be removed, others are the most valuable data points in the entire dataset. The decision requires domain knowledge and business context.

🗑️ Remove the outlier when:
  • → It is clearly a data entry error (age = 999)
  • → It is physically impossible (negative height)
  • → It is a measurement malfunction (sensor spike)
  • → It is irrelevant to the business problem
🔍 Keep the outlier when:
  • → It represents a genuinely rare but real event
  • → It IS the signal you're looking for (fraud, failure, churn)
  • → Removing it would introduce bias
  • → It may carry important business insight

OUTLIER DETECTION METHODS

import numpy as np
from scipy import stats

col = df["salary"]

# ── Method 1: IQR (Best for skewed distributions) ──────────────────────
Q1, Q3 = col.quantile(0.25), col.quantile(0.75)
IQR = Q3 - Q1
lower = Q1 - 1.5 * IQR
upper = Q3 + 1.5 * IQR
outliers_iqr = df[(col < lower) | (col > upper)]
print(f"IQR method: {len(outliers_iqr)} outliers ({len(outliers_iqr)/len(df)*100:.1f}%)")

# ── Method 2: Z-Score (Best for normal distributions) ──────────────────
z_scores = np.abs(stats.zscore(col.dropna()))
outliers_z = df[np.abs(stats.zscore(col.fillna(col.mean()))) > 3]
print(f"Z-score method: {len(outliers_z)} outliers")

# ── Method 3: Visual inspection ────────────────────────────────────────
fig, axes = plt.subplots(1, 3, figsize=(15, 4))

# Histogram with boundaries
axes[0].hist(col, bins=50, color="#5b7cff", alpha=0.8)
axes[0].axvline(lower, color="red", linestyle="--", label=f"Lower: {lower:.0f}")
axes[0].axvline(upper, color="red", linestyle="--", label=f"Upper: {upper:.0f}")
axes[0].legend()
axes[0].set_title("Histogram + IQR Boundaries")

# Boxplot
sns.boxplot(y=col, ax=axes[1], color="#7a4bff")
axes[1].set_title("Boxplot")

# Sorted scatter (shows all values)
axes[2].scatter(range(len(col)), col.sort_values(), alpha=0.3, s=5, color="#28e0ff")
axes[2].set_title("Sorted Values Scatter")
plt.tight_layout()
plt.show()

# ── Treatment Options ───────────────────────────────────────────────────
# Option 1: Remove outliers
df_clean = df[(col >= lower) & (col <= upper)]

# Option 2: Cap (Winsorize) — keeps row but limits extreme values
df["salary_capped"] = col.clip(lower=lower, upper=upper)

# Option 3: Log transform — compresses skewed distributions
df["salary_log"] = np.log1p(col)  # log(1 + x) handles zeros

🌡️ 4.7 Correlation Analysis and Heatmaps

Correlation measures the strength and direction of the linear relationship between two numerical variables. It ranges from −1 (perfect negative) to +1 (perfect positive), with 0 meaning no linear relationship.

+0.8 to +1
Strong Positive

As X increases, Y increases strongly. E.g., experience vs salary.

-0.3 to +0.3
Weak / None

Little to no linear relationship. May still have non-linear correlation.

-0.8 to -1
Strong Negative

As X increases, Y decreases strongly. E.g., price vs quantity sold.

CORRELATION MATRIX AND HEATMAP

import seaborn as sns
import matplotlib.pyplot as plt

# Select only numerical columns
numeric_df = df.select_dtypes(include="number")

# Compute correlation matrix (Pearson is default)
corr_matrix = numeric_df.corr()

# Plot heatmap
plt.figure(figsize=(10, 8))
mask = np.triu(np.ones_like(corr_matrix, dtype=bool))  # Hide upper triangle (redundant)
sns.heatmap(
    corr_matrix,
    mask=mask,
    annot=True,              # Show numbers on cells
    fmt=".2f",               # 2 decimal places
    cmap="coolwarm",         # Blue = negative, Red = positive
    center=0,                # Center colormap at 0
    vmin=-1, vmax=1,
    linewidths=0.5,
    square=True,
    cbar_kws={"shrink": 0.8}
)
plt.title("Correlation Matrix — Numerical Variables", pad=20)
plt.tight_layout()
plt.show()

# Find the strongest correlations (excluding self-correlation)
corr_pairs = (corr_matrix
    .unstack()
    .reset_index()
    .rename(columns={"level_0": "var1", "level_1": "var2", 0: "correlation"})
    .query("var1 != var2")
    .assign(abs_corr=lambda x: x["correlation"].abs())
    .sort_values("abs_corr", ascending=False)
    .drop_duplicates(subset=["abs_corr"])
    .head(10)
)
print(corr_pairs)
⚠️ Correlation ≠ Causation

Ice cream sales and drowning incidents are strongly correlated — not because ice cream causes drowning, but because both increase in summer. Always ask: is there a logical causal mechanism, or is a third variable (a confounding variable) responsible? Also: Pearson correlation only detects linear relationships. Two variables may be strongly related in a non-linear way and show 0 correlation.

⚙️ 4.9 Feature Engineering — Creating Variables That Models Can Learn From

Raw data is rarely in the ideal format for a machine learning model. Feature engineering is the process of using domain knowledge and EDA insights to create, transform, or select variables that better represent the underlying problem.

COMMON FEATURE ENGINEERING TECHNIQUES

import pandas as pd
import numpy as np

# ── 1. Date/Time Features ───────────────────────────────────────────────
df["order_date"] = pd.to_datetime(df["order_date"])
df["year"]        = df["order_date"].dt.year
df["month"]       = df["order_date"].dt.month
df["day_of_week"] = df["order_date"].dt.dayofweek      # 0=Monday, 6=Sunday
df["is_weekend"]  = (df["day_of_week"] >= 5).astype(int)
df["quarter"]     = df["order_date"].dt.quarter

# Days since first purchase (recency)
df["days_since_first"] = (df["order_date"] - df["order_date"].min()).dt.days

# ── 2. Interaction Features ─────────────────────────────────────────────
df["revenue_per_unit"]   = df["revenue"] / df["units_sold"]
df["age_experience_ratio"] = df["age"] / (df["experience"] + 1)

# ── 3. Encoding Categorical Variables ───────────────────────────────────
# Label encoding (for ordinal categories: Low < Medium < High)
size_map = {"Small": 1, "Medium": 2, "Large": 3}
df["company_size_encoded"] = df["company_size"].map(size_map)

# One-Hot Encoding (for nominal categories — no order)
df_encoded = pd.get_dummies(df, columns=["region", "category"], drop_first=True)

# ── 4. Normalization / Scaling ──────────────────────────────────────────
from sklearn.preprocessing import StandardScaler, MinMaxScaler

# Standard Scaler: mean=0, std=1 (for algorithms that assume normality)
scaler = StandardScaler()
df["salary_scaled"] = scaler.fit_transform(df[["salary"]])

# MinMax Scaler: range [0, 1] (for neural networks, KNN)
minmax = MinMaxScaler()
df["salary_normalized"] = minmax.fit_transform(df[["salary"]])

# ── 5. Binning Continuous Variables ─────────────────────────────────────
df["age_group"] = pd.cut(df["age"],
    bins=[0, 25, 35, 50, 65, 100],
    labels=["Under 25", "25-35", "35-50", "50-65", "65+"])

# ── 6. Log Transform (fixing right skew) ───────────────────────────────
df["salary_log"] = np.log1p(df["salary"])   # log(1+x) handles 0 values
df["revenue_log"] = np.log1p(df["revenue"])
📏 When to Scale?
Algorithm Needs Scaling? Why?
Linear/Logistic Regression✅ YesCoefficients are scale-dependent
KNN, SVM✅ YesDistance-based — scale dominates distances
Neural Networks✅ YesGradient descent converges faster
Decision Trees, Random Forest, XGBoost❌ NoTree splits are scale-invariant

📋 4.11 Documenting EDA — The Analyst's Report

An EDA without documentation is just a series of charts. A professional EDA report communicates your findings to others — data engineers, business stakeholders, other scientists — and serves as a reference for all future modelling decisions.

📝 Structure of a Professional EDA Report:
1. Executive Summary — What data? What question? Key findings in 3 sentences.
2. Dataset Overview — Shape, dtypes, memory, source, time range.
3. Data Quality — Missing values %, duplicates, anomalies found, fixes applied.
4. Univariate Analysis — Distribution of each variable. Skewness. Outliers.
5. Bivariate Analysis — Key relationships. What correlates with the target?
6. Multivariate Analysis — Segments, interactions, PCA if applicable.
7. Key Insights — Numbered list of discoveries. What was surprising?
8. Recommendations — Which features to use? What to engineer? What to investigate further?
9. Limitations — What the data cannot answer. Potential biases.

⚠️ 4.12 Common Mistakes in EDA — And How to Avoid Them

🚫 Common Mistakes
  • → Skipping EDA entirely and jumping to modelling
  • → Trusting only the mean — ignoring std dev and distribution shape
  • → Removing all outliers without domain investigation
  • → Making cleaning decisions without documenting them
  • → Confusing correlation with causation
  • → Leaking future data into EDA (target leakage)
  • → Normalizing before the train/test split
  • → Overplotting — using color for every variable
  • → Choosing chart types based on aesthetics, not the data type
  • → Treating categorical NaN the same as numerical NaN
✅ Best Practices
  • → Always start with df.info() and df.describe()
  • → Always work on a copy: df = df_raw.copy()
  • → Document every decision with justification
  • → Compare mean vs median — large difference = skewed data
  • → Visualize before deciding how to treat outliers
  • → Use the right chart for the data type (histogram for continuous, bar for categorical)
  • → Check class balance in classification targets
  • → Scale features only after train/test split
  • → Never use df.plot() alone — always title, axes labels
  • → Ask "so what?" for every chart — if it has no insight, cut it
