🌐 What are Networks and Protocols?

A network is a collection of devices (computers, servers, phones, routers) connected together to exchange information.

Protocols are the rules that define how that communication happens — like a shared language all devices need to "speak" to understand each other.

Understanding networks is essential in cybersecurity: most attacks exploit flaws in communication between devices.

The OSI Model — The 7 Layers

OSI Model

The OSI (Open Systems Interconnection) model divides network communication into 7 layers. Each layer has a specific function and communicates only with adjacent layers. It is the conceptual map that explains how data travels from your keyboard to the server and back.

7 Application User interface. High-level protocols. HTTP, HTTPS, DNS, FTP, SMTP
6 Presentation Data formatting, encryption and compression. TLS/SSL, JPEG, MP4
5 Session Session management and synchronisation between applications. NetBIOS, RPC, cookies
4 Transport Reliable end-to-end data delivery. Error control. TCP, UDP
3 Network Logical addressing and routing between networks. IP, ICMP, routers
2 Data Link Data transfer between nodes on the same local network. Ethernet, MAC address, switches
1 Physical Bit transmission over physical media (cables, wi-fi, fibre). RJ45, fibre optic, radio
💡
Mnemonic (layers 7→1) "All People Seem To Need Data Processing" — Application, Presentation, Session, Transport, Network, Data Link, Physical.
🔐
Why does the OSI model matter for security? Each layer has specific vulnerabilities. Phishing attacks layer 7 (user). DDoS can attack layers 3/4 (network/transport). SQL Injection attacks layer 7 (application). Knowing the layers helps identify where to defend.
TCP/IP — The Real Internet Model

TCP vs UDP

The TCP/IP model is the practical, simplified version of the OSI — it is what actually runs on the internet. The two most important transport protocols are TCP and UDP.

Feature TCP (Transmission Control Protocol) UDP (User Datagram Protocol)
ConnectionConnection-oriented (handshake)Connectionless
ReliabilityHigh — guarantees deliveryNo guarantee
SpeedSlower (acknowledgements)Very fast
Packet orderMaintains orderMay arrive out of order
Typical useWeb, email, SSH, FTPStreaming, gaming, DNS, VoIP
Security riskSYN Flood, session hijackingUDP Flood, amplification attacks

TCP Three-Way Handshake

Before any data is exchanged, TCP establishes a connection through 3 steps:

TCP Three-Way Handshake
💻 Client
🖥️ Server
SYN →
"Can I connect?"
← SYN-ACK
"Yes, you can."
ACK →
"Thanks, starting now."
✅ Connection established — data begins to flow
⚠️
SYN Flood Attack An attacker sends thousands of SYN requests without ever completing the handshake (never sending the final ACK). The server waits for responses that never arrive, exhausting its resources. This is a form of DDoS attack at the transport layer.

IP Spoofing An attacker can falsify the source IP address of their packets to impersonate another device. It is used in DDoS reflection attacks and to bypass network filters. Mitigation: ingress filtering (BCP38), additional authentication.

Ports — The "Apartment" inside the IP Address

If the IP is the building, the port is the apartment. Each service listens on a specific port. There are 65,535 ports — the first 1,024 are reserved for well-known services.

HTTP vs HTTPS & TLS

HTTP vs HTTPS — The difference that matters

HTTP is the base web protocol. HTTPS is the secure version — it encrypts everything with TLS (Transport Layer Security).

HTTP vs HTTPS — What an attacker sees
💻 Browser
──── HTTP ────
🖥️ Server
😈 Attacker sees: "username=admin&password=12345"
💻 Browser
── HTTPS/TLS ──
🖥️ Server
😈 Attacker sees: "x7Kp#@!9mQr..." (unreadable)
### HTTP Request (INSECURE — everything visible as plain text) ###

GET /login HTTP/1.1
Host: example.com
User-Agent: Mozilla/5.0
Content-Type: application/x-www-form-urlencoded

username=admin&password=12345   ← visible to anyone on the network!


### Server response ###

HTTP/1.1 200 OK
Content-Type: text/html
Set-Cookie: session=abc123; HttpOnly

<html>...dashboard page...</html>
Line by line:

GET /login HTTP/1.1 — HTTP method (GET, POST, PUT, DELETE) + path + protocol version.
Host — identifies the destination server (needed when one IP serves multiple domains).
username=admin&password=12345 — over plain HTTP, this data travels in clear text — anyone on the same network can read it with Wireshark.
Set-Cookie: session=abc123 — the server creates a session; HttpOnly prevents JavaScript from accessing the cookie (protection against XSS).
With HTTPS — the entire exchange is encrypted with TLS; an attacker sees only random, unreadable bytes.

FeatureHTTPHTTPS
Default port80443
EncryptionNoneTLS 1.2 / 1.3
Data visibleYes (easy sniffing)No (encrypted)
Certificate requiredNoYes (CA or Let's Encrypt)
SEO / BrowsersMarked as "Not secure"Green padlock ✅
DNS — The Name System

How does DNS work?

DNS (Domain Name System) is the internet's "phone book". It translates human-readable names (google.com) into IP addresses computers understand (142.250.185.78).

DNS Spoofing / Cache Poisoning An attacker can inject fake DNS records into a resolver's cache. When a user tries to access "mybank.com", they are silently redirected to a fake site identical to the original. Mitigation: DNSSEC, HTTPS (the certificate won't match), DNS over HTTPS (DoH).
Firewall, VPN & Wireshark

Firewall — The Network Gatekeeper

A firewall inspects incoming and outgoing traffic and decides what passes through based on defined rules. It is the first line of defence of any network.

Firewall TypeHow it worksExample
Packet FilteringAnalyses headers (IP, port, protocol)iptables, router ACLs
StatefulTracks TCP connection stateWindows Firewall, pf (macOS)
Application Layer (WAF)Analyses HTTP content (layer 7)Cloudflare WAF, ModSecurity
Next-Gen (NGFW)Deep inspection + integrated IDS/IPSPalo Alto, Fortinet
 
# Example iptables rules (Linux)

# Block all incoming traffic by default
iptables -P INPUT DROP

# Allow already established connections
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# Allow SSH (port 22) only from a specific IP
iptables -A INPUT -p tcp --dport 22 -s 192.168.1.10 -j ACCEPT

# Allow HTTPS (port 443) from any source
iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# Block port 23 (Telnet) — completely insecure
iptables -A INPUT -p tcp --dport 23 -j DROP
Line by line:

-P INPUT DROP — default policy: block everything incoming (whitelist approach — more secure).
ESTABLISHED,RELATED — allows responses to connections the server itself initiated.
--dport 22 -s 192.168.1.10 — SSH restricted to one specific IP; avoids internet brute-force.
--dport 443 — HTTPS open to the world (required to serve a website).
--dport 23 -j DROP — Telnet blocked; use SSH instead.

VPN — Secure Tunnel

A VPN (Virtual Private Network) creates an encrypted tunnel between your device and a VPN server. All traffic passes through that tunnel — even on public Wi-Fi, nobody can see what you send.

Wireshark — Viewing Network Traffic

Wireshark is the most widely used tool to capture and analyse network packets. It is essential for investigating attacks, diagnosing problems and understanding what really flows across a network. 

# Most useful Wireshark filters in cybersecurity

# Show only HTTP traffic
http

# Show requests to a specific IP
ip.dst == 192.168.1.1

# Show TCP traffic on port 443 (HTTPS)
tcp.port == 443

# Filter by DNS protocol
dns

# See login attempts (HTTP POST requests)
http.request.method == "POST"

# Find packets with SYN flag (connection start)
tcp.flags.syn == 1

# Detect port scanning (many SYN without ACK)
tcp.flags.syn == 1 && tcp.flags.ack == 0
⚠️
Ethical use of Wireshark Capturing network traffic without authorisation is illegal in most countries. Wireshark should only be used on networks you administer, in lab environments (e.g. TryHackMe), or with explicit written authorisation.

Main Attacks on Networks

AttackOSI LayerHow it worksDefence
Man-in-the-Middle (MitM) 2, 3, 7 Intercepts communication between two points (e.g. ARP poisoning) HTTPS, VPN, HSTS, certificates
ARP Poisoning 2 Sends fake ARP replies to associate attacker's MAC with victim's IP Dynamic ARP Inspection, segmentation

DDoS (SYN Flood) 3, 4 Floods server with SYN requests without completing handshake Rate limiting, SYN cookies, CDN, WAF

Packet Sniffing 1, 2 Captures unencrypted traffic on shared networks End-to-end encryption, switches instead of hubs
🔍
Nmap — The Network Scanner Nmap is the standard tool for discovering hosts and services on a network. A basic scan: nmap -sV -p 1-1000 192.168.1.1 — discovers open ports and service versions. Used by pentesters and system administrators (always with authorisation!).
