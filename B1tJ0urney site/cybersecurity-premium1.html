🛡️ What is Cybersecurity?

Cybersecurity is the set of practices, technologies and processes that protect systems, networks and data against attacks, damage or unauthorised access.

We live in an increasingly digital world — banking, healthcare, transport, communications — everything depends on computer systems. Protecting them has become as important as locking your front door.

The 3 Pillars — CIA Triad

The CIA Triad

All information security rests on three fundamental principles known as the CIA Triad:

🔒
Confidentiality

Only authorised parties can access the information. Ex: password to log into an account.

✅
Integrity

Information cannot be altered without authorisation. Ex: immutable system logs.

⚡
Availability

The system must be available when needed. Ex: a website that cannot afford downtime.

💡
Practical example A ransomware attack violates availability (blocks access) and may violate confidentiality (stolen data). That is why companies perform regular backups.

Passwords & Hashing

How does a Secure Password work?

Passwords should never be stored as plain text in a database. If the database is compromised, all accounts are exposed. The solution is hashing.

# password_hash.py — Hashing example with Python

import hashlib

def create_hash(password):
    """Creates a SHA-256 hash of the given password."""
    hash_result = hashlib.sha256(password.encode()).hexdigest()
    return hash_result

def verify_password(password, stored_hash):
    """Compares the entered password with the stored hash."""
    return create_hash(password) == stored_hash

# Registration flow
original_password = "MyPassword123!"
hash_in_db = create_hash(original_password)
print(f"Stored hash: {hash_in_db}")

# Login flow
attempt = input("Enter your password: ")
if verify_password(attempt, hash_in_db):
    print("✅ Access granted!")
else:
    print("❌ Incorrect password.")
Line by line:

import hashlib — imports Python's built-in cryptographic library.
password.encode() — converts the string to bytes (SHA-256 works with bytes, not text).
hashlib.sha256(...) — applies the SHA-256 algorithm (always produces 64 hex characters).
.hexdigest() — returns the result as a human-readable hexadecimal string.
verify_password() — in practice you never reverse the hash; you hash the attempt and compare.

AlgorithmHash sizeSecure today?Recommended use
MD5128 bits / 32 chars❌ NoChecksums only, never passwords
SHA-1160 bits / 40 chars❌ NoDeprecated since 2017
SHA-256256 bits / 64 chars⚠️ PartialGood for integrity, weak for passwords
bcryptVariable + salt✅ YesIdeal for storing passwords
Argon2Variable + salt✅ YesCurrent best choice for passwords
⚠️
SHA-256 is not enough for passwords! SHA-256 is very fast — an attacker can test billions of combinations per second (brute force). Always use bcrypt or Argon2 for passwords, as they are deliberately slow algorithms.
Common Threats

Main Types of Attacks

Knowing the enemy is the first step to defence. These are the most common attacks:

🎣
Phishing

Fake email/SMS impersonating a legitimate entity to steal credentials.

💣
Malware

Malicious software: viruses, trojans, spyware, ransomware.

🔐
Ransomware

Encrypts your files and demands a ransom (usually in crypto).

🌊
DDoS

Floods a server with traffic to make it unavailable.

🕵️
Man-in-the-Middle

Intercepts communications between two parties without either knowing.

💉
SQL Injection

Inserts malicious SQL code into an input field to manipulate the database.

🎭
Social Engineering

Psychological manipulation to obtain information or access.

🔓
Brute Force

Automatically tests millions of passwords until the correct one is found.

Encryption

Encryption: Symmetric vs. Asymmetric

Encryption transforms readable data (plaintext) into unreadable data (ciphertext) without the correct key. There are two main types:

TypeHow it worksAlgorithmsAdvantageDisadvantage
🔑 SymmetricSame key to encrypt and decryptAES, DES, ChaCha20Very fastHow do you share the key securely?
🗝️ AsymmetricKey pair: public (encrypts) + private (decrypts)RSA, ECC, DSAPublic key can be shared freelyMuch slower
Asymmetric Encryption — HTTPS
🌐 Server
has private key
→ public key →
💻 Browser
→ encrypted data →
🌐 Server
decrypts with private key
ℹ️
HTTPS = HTTP + TLS The TLS protocol uses asymmetric encryption to exchange a session key, then uses symmetric encryption (AES) for the rest of the communication — leveraging the speed advantages of each type. 
# Simple symmetric encryption example with Python
# (install with: pip install cryptography)

from cryptography.fernet import Fernet

# Generate a random key
key = Fernet.generate_key()
cipher = Fernet(key)

# Encrypt
message = b"Confidential data"
encrypted = cipher.encrypt(message)
print(f"Encrypted: {encrypted}")

# Decrypt
original = cipher.decrypt(encrypted)
print(f"Original: {original.decode()}")
What happens here:

Fernet.generate_key() — generates a random AES-128 key of 256 bits encoded in base64.
cipher.encrypt() — encrypts the message; the result differs each time (due to random IV).
cipher.decrypt() — only works with the same key; without it, data is unreadable.
b"..." — byte strings; Fernet works with bytes, not regular strings.

Two-Factor Authentication (2FA)

2FA — An essential extra layer

Two-factor authentication requires the user to prove their identity with two distinct elements. Even if a password is stolen, the attacker cannot access the account without the second factor.

FactorTypeExampleSecurity
Something you knowKnowledgePassword, PIN, secret questionMedium
Something you havePossessionPhone (TOTP), hardware key (YubiKey)High
Something you areBiometricFingerprint, Face ID, irisHigh
Login Flow with 2FA (TOTP)
📧 Email + Password
→
✅ 1st Factor OK
→
📱 TOTP Code (6 digits)
→
🔓 Access Granted
💡
TOTP — Time-based One-Time Password Apps like Google Authenticator or Authy generate a 6-digit code that changes every 30 seconds, based on an algorithm (HMAC-SHA1) and the current time. Even if someone intercepts the code, it expires in 30 seconds.
Essential Best Practices

Digital Hygiene — What you should ALWAYS do

PracticeWhy it mattersHow to implement
✅ Unique passwords per serviceIf one service is compromised, others remain protectedUse a password manager (Bitwarden, 1Password)
✅ Enable 2FA everywhereStolen password = account still protectedGoogle Authenticator, Authy, YubiKey
✅ Regular updatesPatches fix known vulnerabilitiesEnable automatic updates
✅ Backups (3-2-1 rule)Protection against ransomware and failures3 copies, 2 different media, 1 off-site
✅ VPN on public Wi-FiPrevents data interception (MitM)Mullvad, ProtonVPN, WireGuard
✅ Check HTTPSEnsures the connection is encryptedLook for 🔒 in the browser address bar
✅ Don't click suspicious linksMain phishing vectorVerify sender, hover over link before clicking
🚨
Golden rule If you receive an urgent message asking for credentials, money or access — even if it appears to be from your bank, boss or Google — stop, breathe and verify through another channel (call, go to the website directly). Urgency is the main tactic of social engineering.
